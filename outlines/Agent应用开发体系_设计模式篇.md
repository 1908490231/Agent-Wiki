# Agent应用开发体系 - 设计模式篇

> 本篇聚焦 Agent 系统的经典范式、架构模式与企业级应用架构，帮助开发者理解和选择合适的 Agent 设计方案。
>
> **不包含**：具体框架用法（见工程篇）、模型理论（见模型算法篇）、提示词技巧（见 Prompt Engineering 篇）

## 一、Agent 经典范式

### 1.1 ReAct 范式
- 核心思想：Thought → Action → Observation 循环
- 推理（Reasoning）与行动（Acting）交替执行
- 工作流程
    1. 接收任务
    2. Thought：分析当前状态，决定下一步
    3. Action：执行工具调用或操作
    4. Observation：观察执行结果
    5. 循环直至任务完成
- 适用场景：需要与外部环境交互的复杂任务
- 优劣势分析

### 1.2 Plan-and-Solve / Plan-and-Execute
- 核心思想：先规划后执行，分离计划与执行阶段
- 工作流程
    1. 理解任务目标
    2. 生成分步计划
    3. 逐步执行计划
    4. 根据执行结果动态调整计划
- 与 ReAct 的对比
- 适用场景：多步骤复杂任务、需要全局规划的场景

### 1.3 Reflection（反思）
- 核心思想：Agent 对自身输出进行评估与改进
- 工作流程
    1. 生成初始输出
    2. 自我评估输出质量
    3. 识别问题与改进方向
    4. 重新生成改进后的输出
- 反思策略：自我批评、外部评估、对比反思
- 适用场景：代码生成、文本写作等需要迭代优化的任务

## 二、单 Agent 模式

### 2.1 Tool Agent（工具代理）
- 核心概念：LLM + 工具调用能力
- 工具选择策略
- 工具调用链
- 错误处理与重试

### 2.2 Code Agent（代码代理）
- 核心概念：通过生成和执行代码完成任务
- 代码沙箱环境
- 代码执行安全控制
- 典型实现：OpenAI Code Interpreter

### 2.3 Conversational Agent（对话代理）
- 核心概念：多轮对话驱动的交互式 Agent
- 对话状态管理
- 上下文维护
- 意图识别与槽位填充

### 2.4 RAG Agent
- 核心概念：结合检索增强生成的智能 Agent
- 自适应检索策略
- 检索-生成-验证循环
- 适用场景：知识密集型问答

### 2.5 Autonomous Agent（自主代理）
- 核心概念：自主设定子目标并执行的 Agent
- 目标分解与规划
- 长期记忆管理
- 自我反思与进化
- 典型实现：AutoGPT、BabyAGI

## 三、多 Agent 协作模式

### 3.1 Master-Worker 模式
- 架构：一个主 Agent 分配任务，多个 Worker Agent 执行
- 任务分解与分配策略
- Worker 特化设计
- 结果汇总与整合
- 适用场景：可并行化的复杂任务

### 3.2 Handoff 模式（任务交接）
- 架构：Agent 之间按条件路由和交接任务
- 交接条件设计
- 上下文传递
- 回退机制
- 典型实现：OpenAI Agents SDK Handoff

### 3.3 群聊模式（GroupChat）
- 架构：多个 Agent 在共享对话中协作
- 发言顺序策略（轮询、管理员选择、自由发言）
- 角色分工与协作规则
- 共享记忆/黑板机制
- 典型实现：AutoGen GroupChat

### 3.4 辩论/竞争模式
- 架构：多个 Agent 对同一问题提出不同方案
- 观点生成与评估
- 投票/裁判机制
- 共识达成策略
- 适用场景：需要多角度分析的决策任务

### 3.5 管道模式（Pipeline）
- 架构：Agent 按顺序处理任务的不同阶段
- 阶段定义与输入输出规范
- 错误传播与处理
- 适用场景：有明确阶段划分的流水线任务

### 3.6 层级模式（Hierarchical）
- 架构：多层级的 Agent 树状组织
- 层级间的通信与控制
- 递归委托
- 适用场景：大规模复杂系统

## 四、企业级架构模式

### 4.1 知识密集型场景架构（企业 AI 应用平台）
- 六层架构
    - ⓪ 基础设施层（混合云部署）：公有云 SaaS / 私有化部署 / 混合云模式
    - ① 平台基础层（企业级 PaaS）：多租户管理、权限中心、审计日志、监控告警、配置中心、服务网格、消息队列、对象存储
    - ② 底座模型层：模型管理中枢（多模型路由、负载均衡、成本优化、降级策略）+ 数据治理中心（数据分级、脱敏加密、合规审计）
    - ③ AI 底座层（核心引擎）：文档解析引擎、RAG 检索引擎、Agent 编排引擎、内容生成引擎、逻辑校验引擎、数值计算引擎、流程引擎、多模态引擎
    - ④ 场景模板层：行业知识库、审查规则库、输出模板库、工作流编排
    - ⑤ 应用层（Agent 应用市场）：合同审查、方案审查、标书编写、知识问答、数字员工、智能客服等
- 适用场景：大型企业的知识管理与业务自动化
- 关键设计要点

### 4.2 自主代理型架构（通用 Agent 底座引擎）
- 六层架构
    - ⓪ 基础设施层（部署环境）：本地开发 / 服务器部署 / 云端部署
    - ① 平台基础层（原子工具 + 安全管控）：文件工具、搜索工具、系统工具、安全管控、配置中心、日志系统
    - ② 模型层（模型管理 + 记忆系统）：多模型路由、OpenAI 兼容接口、记忆存储（经验/轨迹/黑板协同）
    - ③ AI 引擎层（核心能力）：Agent 引擎（ReAct 循环）、Handoff 引擎（Master-Worker）、反思引擎（经验蒸馏）、引导引擎、工具执行引擎、动作解析引擎、压缩引擎、中断引擎
    - ④ 技能层（可插拔 Skill 系统）：Skill 包管理、配置驱动引擎、执行器扩展、参数校验
    - ⑤ 应用层（场景化 Agent）：文档审查、内容写作、简历定制、会议分析、文件整理等
- 核心架构亮点
    - ReAct 推理循环
    - Master-Worker 任务交接
    - 可插拔 Skill 系统
    - 多模型统一接口
    - 反思进化能力
    - 安全管控体系
- 数据流向
    - 用户输入 → AgentEngine → ModelFactory → ActionParser → ToolExecutor → HandoffTool → Reflector → 输出
- 适用场景：通用型智能助手、自动化工具平台

### 4.3 数据流与协同模式
- Agent 系统数据流设计
- 状态机与工作流引擎
- 事件驱动架构
- 消息总线与发布/订阅

## 五、模式选型指南

### 5.1 按任务复杂度选型
| 复杂度 | 推荐模式 | 说明 |
|--------|---------|------|
| 简单 | Tool Agent | 单步工具调用即可完成 |
| 中等 | ReAct / Plan-and-Solve | 需要多步推理与执行 |
| 复杂 | Multi-Agent (Master-Worker) | 需要任务分解与并行执行 |
| 极复杂 | 层级多 Agent + 反思 | 大规模系统级任务 |

### 5.2 按场景选型
| 场景 | 推荐模式 | 说明 |
|------|---------|------|
| 客服问答 | Conversational + RAG | 多轮对话 + 知识检索 |
| 文档处理 | Pipeline + Tool Agent | 解析→处理→输出 |
| 代码开发 | Code Agent + Reflection | 生成→执行→反思→改进 |
| 企业业务 | 知识密集型架构 | 多层级企业平台 |
| 通用助手 | 自主代理型架构 | 可扩展的 Agent 底座 |
