## 一、提示词基础

### 1.1 提示词的基本结构
- 系统提示词（System Prompt）
- 用户提示词（User Prompt）
- 助手提示词（Assistant Prompt）
- 角色定义与行为约束
- 输出格式指定

### 1.2 零样本提示（Zero-shot Prompting）
- 基本原理：不提供示例，直接给出任务指令
- 适用场景
- 技巧：清晰指令、角色设定、输出约束

### 1.3 少样本提示（Few-shot Prompting）
- 基本原理：提供示例引导模型输出
- 示例选择策略
- 示例数量与质量的权衡
- 示例格式设计

### 1.4 指令工程（Instruction Engineering）
- 指令编写原则：清晰、具体、无歧义
- 结构化指令设计
- 约束条件表达
- 负面指令（告诉模型不要做什么）

### 1.5 系统提示词设计
- 系统提示词的作用与边界
- 角色定义最佳实践
- 行为规则设计
- 输出格式约束
- 安全护栏（Guardrails）嵌入
- 系统提示词模板

## 三、进阶推理技巧

### 3.1 思维链（Chain-of-Thought, CoT）
- 基本原理：引导模型展示推理过程
- 零样本 CoT："让我们一步一步思考"
- 少样本 CoT：提供推理示例
- 自动 CoT（Auto-CoT）
- 适用场景：数学推理、逻辑分析、复杂决策

### 3.2 思维树（Tree-of-Thought, ToT）
- 基本原理：多条推理路径并行探索
- 搜索策略：BFS、DFS
- 回溯机制
- 评估与剪枝
- 适用场景：创意写作、复杂规划

### 3.3 思维图（Graph-of-Thought, GoT）
- 基本原理：图结构的推理网络
- 多路径融合
- 与 ToT 的对比

### 3.4 自我一致性（Self-Consistency）
- 基本原理：多次采样取多数一致的结果
- 采样策略（温度设置）
- 投票机制
- 适用场景：提高推理可靠性

### 3.5 ReAct 提示模式
- 推理 + 行动交替的提示模板设计
- Thought / Action / Observation 的格式规范
- 与 Agent 框架的结合
- 提示词模板示例

### 3.6 结构化输出引导
- JSON 模式输出
- XML/Markdown 格式输出
- Schema 约束
- 强制格式遵循技巧
- Function Calling 与结构化输出的关系
